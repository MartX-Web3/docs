The Echo Gateway is configured via environment variables. You can provide them in a `.env` file in your working directory, as inline shell exports, or via the `env` block in your Claude Desktop MCP config.

---

## Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `SEPOLIA_RPC_URL` | Yes | — | HTTP RPC endpoint for Sepolia. Alchemy and Infura both work. |
| `PIMLICO_API_KEY` | Yes | — | Your Pimlico bundler API key. Used to submit UserOperations. |
| `KEYSTORE_PASSPHRASE` | Yes | — | Passphrase used to encrypt execute and session keys at rest (AES-256-GCM). |
| `GATEWAY_PORT` | No | `3000` | Port the HTTP server and dashboard listen on. |
| `CHAIN_ID` | No | `11155111` | EVM chain ID. `11155111` = Sepolia. Mainnet IDs will be supported at launch. |
| `WALLETCONNECT_PROJECT_ID` | No | Echo built-in | WalletConnect Cloud project ID. Use Echo's built-in ID for development or get your own at [cloud.reown.com](https://cloud.reown.com). |
| `LOG_LEVEL` | No | `info` | Logging verbosity. Accepts `error`, `warn`, `info`, `debug`, `trace`. |
| `KEYSTORE_DIR` | No | `~/.echo/keystore` | Directory where the encrypted keystore file is stored. |

## Variable details

### SEPOLIA_RPC_URL

The gateway uses this endpoint for:
- Reading on-chain policy state
- Estimating UserOp gas
- Checking delegation activation status (`eth_getCode`)
- Querying token balances

Get a free endpoint at [alchemy.com](https://alchemy.com) (recommended) or [infura.io](https://infura.io).

```bash
# Alchemy format
SEPOLIA_RPC_URL=https://eth-sepolia.g.alchemy.com/v2/your-api-key

# Infura format
SEPOLIA_RPC_URL=https://sepolia.infura.io/v3/your-project-id
```

### PIMLICO_API_KEY

Pimlico is the ERC-4337 bundler that submits UserOperations to the network. The gateway sends signed UserOps to Pimlico's bundler endpoint.

Get a free API key at [dashboard.pimlico.io](https://dashboard.pimlico.io). The free tier supports up to 100 UserOps per day, which is plenty for development.

### KEYSTORE_PASSPHRASE

This passphrase encrypts the execute key and session keys stored on disk. The encryption scheme is AES-256-GCM with a key derived via scrypt.

**Choose a strong passphrase.** It is the only thing protecting your keys at rest. If an attacker obtains both your keystore file and this passphrase, they can sign UserOps on your behalf (up to your policy limits).

The passphrase is never sent to any remote service.

### GATEWAY_PORT

The gateway exposes:
- `http://localhost:{PORT}/` — Web dashboard
- `http://localhost:{PORT}/mcp` — MCP HTTP transport (for advanced integrations)

Change this if port 3000 conflicts with another local service.

### WALLETCONNECT_PROJECT_ID

Echo uses WalletConnect v2 for the EIP-7702 activation flow and dashboard wallet connections. For production use, register your own project at [cloud.reown.com](https://cloud.reown.com) to avoid rate limits on the shared built-in ID.

## Example .env file

```bash .env
# Required
SEPOLIA_RPC_URL=https://eth-sepolia.g.alchemy.com/v2/abc123xyz
PIMLICO_API_KEY=pim_abc123xyz
KEYSTORE_PASSPHRASE=correct-horse-battery-staple-etc

# Optional
GATEWAY_PORT=3000
CHAIN_ID=11155111
LOG_LEVEL=info
WALLETCONNECT_PROJECT_ID=your_wc_project_id
KEYSTORE_DIR=/home/user/.echo/keystore
```

## Claude Desktop MCP configuration

To use Echo as an MCP server with Claude Desktop, add the following to your Claude Desktop config file:

- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json claude_desktop_config.json
{
  "mcpServers": {
    "echo-gateway": {
      "command": "npx",
      "args": ["-y", "@echo-protocol/gateway"],
      "env": {
        "KEYSTORE_PASSPHRASE": "your-passphrase",
        "PIMLICO_API_KEY": "your-pimlico-api-key",
        "SEPOLIA_RPC_URL": "https://eth-sepolia.g.alchemy.com/v2/your-key",
        "GATEWAY_PORT": "3000"
      }
    }
  }
}
```

<Note>
  When running via Claude Desktop's `npx` command, the gateway starts as a subprocess. The dashboard will be accessible at `http://localhost:3000` while Claude Desktop is open. Environment variables in the `env` block override any `.env` file.
</Note>

### Using a globally installed gateway

If you have installed the gateway globally (`npm install -g @echo-protocol/gateway`), you can reference the binary directly:

```json
{
  "mcpServers": {
    "echo-gateway": {
      "command": "gateway",
      "args": ["start"],
      "env": {
        "KEYSTORE_PASSPHRASE": "your-passphrase",
        "PIMLICO_API_KEY": "your-pimlico-api-key",
        "SEPOLIA_RPC_URL": "https://..."
      }
    }
  }
}
```

## Keystore file

The gateway stores encrypted key material in `KEYSTORE_DIR` (default `~/.echo/keystore`). The directory contains:

| File | Contents |
|---|---|
| `instance.json` | Active instance ID and EOA address (not sensitive) |
| `execute-key.enc` | AES-256-GCM encrypted execute private key |
| `sessions/` | Directory of encrypted session keys, one file per active session |

<Warning>
  Back up `~/.echo/keystore` and your `KEYSTORE_PASSPHRASE` securely. If you lose both, you will need to create a new instance and re-register on-chain. Your funds are never at risk — only the gateway's signing capability is lost.
</Warning>
